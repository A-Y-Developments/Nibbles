1 · Apply the consents migration 2 min

NIB-145 added a new consents table. Until you apply it, the in-app consent gate still works but the receipts won't persist (failures are silent + logged to Crashlytics).

Option A — Push via CLI (recommended)

From the project root: 
cd /Users/adithyafp_/Projects/nibbles
supabase link --project-ref tysvrlxvaxkucdgbgfpc
supabase db push
The CLI will prompt for the database password — it's in .env.dev as SUPABASE_DB_PASSWORD.

Option B — Dashboard SQL editor

Open the SQL editor and paste the migration body:
Open SQL editor ↗
CREATE TABLE IF NOT EXISTS consents (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  baby_id UUID REFERENCES babies(id) ON DELETE CASCADE,
  consent_type TEXT NOT NULL CHECK (consent_type IN ('solids_introduction', 'under_6mo_responsibility')),
  given_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

ALTER TABLE consents ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users insert own consents" ON consents
  FOR INSERT TO authenticated
  WITH CHECK (user_id = auth.uid());

CREATE POLICY "users read own consents" ON consents
  FOR SELECT TO authenticated
  USING (user_id = auth.uid());

CREATE INDEX consents_user_id_idx ON consents (user_id);

Verify

In the SQL editor: 
SELECT * FROM consents LIMIT 1;
Should return (0 rows) with no error.

3 · Apple sign-in setup ~25 min

Apple's setup is a bit more clicky. You'll: enable the capability on your App IDs, create a Sign in with Apple Key (.p8), and create a Services ID for the Supabase callback.

3.1 — Enable capability on App IDs

Open Identifiers → find com.aydev.nibbles → Edit → check ✅ Sign in with Apple → Save.

Open Identifiers ↗

Repeat for com.aydev.nibbles.dev.

If the dev App ID doesn't exist yet, create it with the same name + register the dev Bundle ID.

3.2 — Create the Sign in with Apple Key (.p8)

Go to Keys → + → name "Nibbles SIWA Key" → check ✅ Sign in with Apple → Configure → Primary App ID = com.aydev.nibbles → Save → Continue → Register.

Open Keys ↗
⚠️ You can only download the .p8 file ONCE. Save it somewhere safe (1Password, drive, etc.) Note the Key ID on the same page — it's a 10-character alphanumeric string.

Note your Team ID.

It's in the top-right of the Apple Developer portal, next to your name. 10 characters.

3.4 — Configure Xcode signing

Open ios/Runner.xcworkspace in Xcode → select the Runner target → Signing & Capabilities.

This project uses build configurations for flavors (not separate targets), so you'll see tabs along the top: All, Debug, Release, Profile, Debug-dev, Release-dev, Debug-prod, Release-prod, etc.

Click the All tab and verify Team is set to your Apple Developer team. If it shows "Multiple values" or any tab is blank, click each config and pick the team manually.

Also verify the Bundle Identifier per config:
